# RocketMQ 订单超时取消功能 - 测试指南

## 前置条件

1. 确保 RocketMQ 服务已启动
   ```bash
   # 启动 NameServer
   sh mqnamesrv
   
   # 启动 Broker（启用自动创建Topic）
   sh mqbroker -n localhost:9876 autoCreateTopicEnable=true
   ```

2. 确保应用配置正确
   - 检查 `application.yaml` 中的 RocketMQ 配置
   - 确认 NameServer 地址正确：`172.17.156.101:9876`

3. 启动 DXShop-order 服务

## 测试步骤

### 测试1：快速验证（1分钟延迟）

#### 方法1：使用 curl 命令
```bash
curl -X POST http://localhost:8080/test/mq/generate-test \
  -H "Content-Type: application/json"
```

#### 方法2：使用 Postman
- URL: `POST http://localhost:8080/test/mq/generate-test`
- 无需请求体
- 点击 Send

#### 预期结果
1. 立即看到日志：
   ```
   订单超时检查延时消息发送成功, orderSn: xxx, msgId: xxx, delayLevel: 5
   ```

2. 等待1分钟后看到日志：
   ```
   收到订单超时检查消息, orderSn: xxx, userId: 1, shopId: 1
   订单不存在，跳过超时处理, orderSn: xxx
   ```
   （因为这是测试消息，订单实际不存在）

### 测试2：完整流程测试（创建真实订单）

#### 步骤1：创建订单
使用现有的订单创建接口创建一个订单（不支付）

#### 步骤2：观察日志
应该看到：
```
订单超时检查延时消息已发送, orderSn: xxx, 将在30分钟后检查支付状态
```

#### 步骤3：等待30分钟
30分钟后，消费者会自动检查订单状态并取消未支付的订单

#### 预期结果
如果订单未支付，会看到：
```
收到订单超时检查消息, orderSn: xxx, userId: xxx, shopId: xxx
订单状态已更新为已取消, orderId: xxx, orderSn: xxx
订单缓存已删除, orderId: xxx
订单超时自动取消成功, orderSn: xxx
```

### 测试3：已支付订单不会被取消

#### 步骤1：创建订单并支付
1. 创建订单
2. 调用支付接口完成支付

#### 步骤2：等待30分钟
30分钟后消费者检查订单

#### 预期结果
```
收到订单超时检查消息, orderSn: xxx, userId: xxx, shopId: xxx
订单已支付，无需取消, orderSn: xxx, payStatus: 1
```

如果使用 RocketMQ Console，可以：
1. 查看 Topic 列表：确认 `TOPIC_ORDER` 已创建
2. 查看消息轨迹：追踪延时消息的发送和消费
3. 查看消费者状态：确认 `order-timeout-consumer-group` 在线

## 常见问题排查

### 问题1：消息发送失败
**现象**：日志显示 "订单超时检查延时消息发送失败"

**排查步骤**：
1. 检查 RocketMQ 服务是否运行
   ```bash
   ps -ef | grep rocketmq
   ```

2. 检查网络连接
   ```bash
   telnet 172.17.156.101 9876
   ```

3. 检查配置文件中的 NameServer 地址是否正确

### 问题2：消费者没有收到消息
**现象**：30分钟后没有任何日志输出

**排查步骤**：
1. 检查消费者是否正常启动
   ```bash
   grep "OrderTimeoutConsumer" ./logs/DXShop-order.log
   ```

2. 检查 RocketMQ Console 中消费者组是否在线

3. 检查 Topic 和 Tag 是否匹配
   - Producer 发送：`TOPIC_ORDER:TAG_ORDER_TIMEOUT`
   - Consumer 监听：相同配置

### 问题3：订单没有被取消
**现象**：收到消息但订单状态未改变

**排查步骤**：
1. 检查订单当前状态
   ```sql
   SELECT order_status, pay_status FROM user_order WHERE order_sn = 'xxx';
   ```

2. 检查日志中的判断逻辑
   - 订单可能已经支付
   - 订单可能已经被取消
   - 订单状态不是待支付
